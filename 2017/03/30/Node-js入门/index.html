模块

　　在编写每个模块时，都有require、exports、module三个预先定义好的变量可供使用

  1. require：在当前模块中加载和使用别的模块，例如：
    var foo1 = require('./foo');

  2. exports：是当前模块导出对象，用于导出模块中的公有方法和属性，例如：

  3. module：将导出对象由默认的普通对象改为一个函数

　　所有的模块在执行过程中只初始化一次，通过命令行参数传递给NodeJS以启动程序的模块被称为主模块。主模块负责调度组成整个程序的其它模块完成工作。

package.json

　　每个项目的根目录下面，一般都有一个package.json文件，定义了这个项目所需要的各种模块，以及项目的配置信息（比如名称、版本、许可证等元数据）。
　　package.json文件内部就是一个JSON对象。npm install命令根据这个配置文件，自动下载所需的模块，也就是配置项目所需的运行和开发环境。

文件编码

　　由于文本编码有UTF8和GBK两种，并且UTF8文件还可能带有BOM。因此在读取不同编码的文本文件时，需要将文件内容转换为JS使用的UTF8编码字符串后才能正常处理。

  1. BOM移除
    BOM共有三位：0xEF、0xBB、0xBF，位于文档的最前面，所以移除BOM字符

  2. GBK->UTF-8
    return iconv.decode(bin, 'gbk');

  3. 单字节编码
    直接可以不管是UTF8还是GBK了。
